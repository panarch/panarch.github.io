Introducing GlueSQL

August 26, 2020 - gluesql websql rust webassembly sql sql-database

GlueSQL!

It's an open-source SQL database fully written in Rust!
Written in Rust, pure functional execution layer, easily swappable storage and web assembly support!

Summary

  1. GlueSQL provides SQL parser & execution layer as a library

You can use GlueSQL itself or to make your own SQL database.

  • SQL parser used: sqlparser-rs
  • Default storage engine: sled
  1. The whole execution layer codes are pure functional

Rust is so good for functional programming, there is no mut in entire GlueSQL excution layer codes.

Verbose version

  1. GlueSQL provides SQL parser & execution layer as a library

Let's just divide SQL database into 4 elements like these below,

  • SQL Parser
  • Execution Layer
  • Storage Layer
  • Management Tools (CLI, connecting interfaces, ...)

In the past, it was very natural to re-implement all that layers when someone wants to make a new SQL database.
Yeah, it was ok, in previous ages..
There were only a few number of SQL database implementations before.

But how about nowadays?
SQL, SQL, SQL everywhere...

We use SQL database when we make server side application.
We also use SQL database when we make complicated frontend application.
Big data analysis? They also provide SQL syntax support.
SQL database is not good for horizontal scaling?
No! Thanks to Raft, now we don't need to worry about manual sharding, something bothersomes anymore.

Still it makes sense to re-implement all that layers when we make a new SQL database?
I don't think so.

Recently, what we are mainly focusing is "Storage Layer".
Though there already exists lots of SQL database implementations but the reason we still make new ones are mainly because of "storage".
Specialize for big data analysis, streaming data, or horizontal scalability, ...

It's not usually because of SQL parser or execution layer.

Then...

Why not making them as a re-usable library?
That was my main motivation to start this project.
(Above is a cool version, my actual motivation was may be... I just wanted to make something using Rust.)

GlueSQL provides very simple way to make custom storage.
Using Rust traits!
What you only need to do is implementing a few store traits, for now it is only two traits: Store, StoreMut

  1. The whole execution layer codes are pure functional

Using database, we store something.
It's very obvious, it is stateful, and very mutable.
So at least for Storage Layer, it's unavoidable not to use mutables.
Then in the case of SQL Parser and Execution Layer?

At first, it's ok to implement SQL parser functionally.
Role of SQL parser is, converting raw SQL string into AST.
input and output are very clear, so it looks ok.
Then how about execution layer?
It's not that different, execution layer is just in between parser and storage layers.
Wow, input and output are also very clear like SQL parser.
There's no need of storing something in execution layer, theoretically.
Role of execution layer is just delivering message properly from parser to storage layer.

And Rust provides very straight forward way to write pure functional codes.
What we only need to is forgetting what mut keyword is.
Then it's all ok, only issue is.. it might be little bit more painful to write code at first.
But that pain is worth enough to pay, then we can see side-effect free and easily debuggable codes.

  1. Web Assembly Support

It may be not directly related to GlueSQL itself.
But... as you know... Rust provides very good WebAssembly supports, and GlueSQL is entirely written in Rust.
There no way for me not to make WebAssembly build, it attracted me toooo much.
So I did it, here is it.

Let's just begin with this demo

  • GlueSQL-js Playground

That demo is not something connecting to remote databases, SQL databse itself runs in web browsers!
It currently provides 3 storage options:

  • Memory
  • localStorage
  • sessionStorage

You can see the code in this repo: gluesql-js-demo
One more interesting stuff, demo uses GlueSQL itself for state management which is often taken by Redux.

Main repo is this: GlueSQL-js
